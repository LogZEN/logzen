Welcome to pyLogView

pyLogView is a next-generation web-based syslog viewer. It currently supports displaying and searching through syslog messages stored in a postgresql database. A dashboard allows you to display nice-looking statistics as well as the latest events.

Initial configuration

Before you will be able to use this software, an initial configuration is needed. PyLogView uses configuration files to store it's settings and user information. The syntax of these configuration files is very simple. They can have multiple sections where each section can have multiple Key-Value pairs. Comments start with a semicolon and empty lines are ignored. 

[section]                     ← a section
key = value                   ← a key-value pair
bla = blubb   ; comment       ← a key-value pair followed by a comment
;comment                      ← a comment

To get started have a look at the config directory inside your pyLogView installation. The configuration files you will find there are:

cherrypy.conf Webserver settings - the address and port number to listen on
logview.conf pyLogView configuration - syslog backend configuration
users.conf pyLogView users - users that are allowed to login

Adding a user

Before you are able to login, you must create a user. Users are stored in the configuration file users.conf. The name of a user must be set as a section name. The only key-value pair the section owns is the users password.

Passwords are stored in form of a sha256 hash. You can use the user creation form to generate a user section and hashed password.

Starting over

After you have finished the configuration, set configured = True in your logview.conf configuration file and restart the application.
